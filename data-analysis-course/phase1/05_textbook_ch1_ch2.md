## 本編

### セクション1：教科書Ch1 — データ分析とは何か

教科書が示すデータ分析の目的：**「意思決定を支援すること」**

```
問いを立てる → データを集める → 分析する → 解釈する → 伝える
```

データ分析は「答えを出す」だけでなく、**「なぜそうなるのか」を説明できること**が重要。

### セクション2：教科書Ch2 — 分析プロセスの全体像

教科書のステップと今回のTitanic作業の対応表：

| 教科書のステップ | Titanic作業 | 対応コマ |
|-----------------|-------------|---------|
| 問いを定義する | 「誰が生き残ったか？」 | コマ2 EDA前 |
| データを理解する | `describe()` ・欠損確認・可視化 | コマ2 EDA |
| データを整える | 欠損補完・エンコーディング | コマ3 前処理 |
| モデルを作る | RF・GBM等で学習 | コマ4 モデル |
| 結果を評価する | CV・Kaggle LBスコア | コマ4 モデル |
| 伝える | （Phase 5で実施） | コマ37〜38 |

### セクション3：「問い」が分析を決める

- **良い問い**：測定できる、答えが意思決定に役立つ
- **悪い問い**：曖昧すぎる、データで答えられない

> Titanicの場合：「誰が生き残ったか」→「性別・客室クラス・年齢が生存に影響する」という**仮説**に落とし込んだ
