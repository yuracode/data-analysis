# 考え中：Phase3 再設計メモ（Udemy → FitLoop 深掘り）

> これは確定版ではなく**考え中のメモ**。流れと設計思想を掴むためのもの。
> コマ単位の完成資料はこの方針が固まってから別途生成する。

---

## 0. 何を変えるのか（前提整理）

- 元のPhase3（コマ14〜28）＝**Udemyで手法を汎用的に学ぶ**補足資料フェーズ → **キャンセル**
- 代わりに **Phase2のFitLoop通し課題を「計画」から「実行」へ進める**フェーズにする
- Phase2終了時点での学生の手元にあるもの：
  - `analysis_plan.md`（分析計画書＝**設計図。まだ実行していない**）
  - `intermediate_report.ipynb`（Q03セグメント＋Q04時系列/A-Bの**着手レベル**）
- つまりPhase2は「こう分析します」という**約束**で終わっている。
  Phase3は **その約束を本物の分析として果たすフェーズ**にする。

### ビフォー／アフター

| | 元のPhase3（Udemy） | 新Phase3（FitLoop実行） |
|---|---|---|
| 主役 | 動画の手法 | 学生自身の問い |
| 手法の学び方 | 動画を見る→使えるか確認 | 問いを解くのに必要だから学ぶ |
| 教材 | Udemy + チェック欄 | FitLoopデータ + 手法カタログ（自分で引く） |
| 成果物 | チェックリスト | 実行済み分析レポート＋ROI再計算 |
| テキスト | ほぼ不使用 | **必要な節だけピンポイント参照** |

## 1. 設計思想（これだけは外さない）

1. **「手法を学ぶ」のではなく「問いに答えるために手法を取りに行く」**
   - 経験者（sklearn/TF済）なので、手の動かし方は教えない。
   - 教えるのは「**この問いにはこの手法**」という対応付けの判断と、**結果の意味の読み方**。

2. **学生主導 = 手順書を渡さない**
   - コマ資料には「今週解く問い」「使えそうな手法カタログ（選択肢）」「詰まったら見るヒント」だけ。
   - 完成コードは載せない。**手法の選択と実装は学生が決める**。

3. **計画書は生き物**
   - 実データを見て計画と違ったら、計画書を**書き換える**のが正解。
   - 「変えた」より「**なぜ変えたか書いた**」を評価する（Phase2の評価軸を継承）。

4. **常にROI・財務に戻る**
   - 月次予算40万円・プラン別月額の制約を毎コマ思い出させる（Phase2踏襲）。
   - Phase3の最後は「計画段階のROI」を「**実データで再計算したROI**」に更新する。

---

## 2. テキストの絞り方（重要）

元Phase3はテキスト不使用だった。新Phase3でも**全面採用はしない**。
「分析の手は動かせるが、考え方の言語が足りない」場面だけ、**節単位で差し込む**。

| 差し込みどころ | 参照する節（候補） | 何のために読むか |
|---|---|---|
| 差は本物か？を扱う回 | Ch8 統計的検定の「考え方」だけ | p値・有意差の意味を**言葉で**押さえる |
| 解釈・So Whatの回 | Ch7 解釈の章の冒頭だけ | 「数字→意味」の翻訳の型 |
| （それ以外） | **読ませない** | 手を動かす時間を確保 |

> 方針：テキストは「読書課題」ではなく「**詰まったときの辞書**」として位置づける。
> 1回の参照は1〜2節・10分以内。Ch6〜8の本格的な回収はPhase4に任せる。

## 3. 全体の流れ（コマ14〜28 / 15コマ）

Phase2で立てた問いを、**4つの探索スプリント**で実行していくイメージ。

```
スプリント0  着地点の再定義        コマ14
スプリント1  現状を正しく描く      コマ15-17  （記述統計・可視化）
スプリント2  その差は本物か        コマ18-21  （前処理・検定・バイアス）
スプリント3  予測して、説明する    コマ22-26  （分類/回帰・評価・SHAP）
スプリント4  計画を結論に変える    コマ27-28  （統合・ROI再計算・発表準備）
```

各スプリントの問いは、Phase2の `analysis_plan.md` の章とそのまま対応する。

## 4. スプリント別メモ

### スプリント0：着地点の再定義（コマ14）
- 計画書を開き直し、「**Phase3終了時に何が言えていれば勝ちか**」を1文で再定義。
- pandasは「講義」ではなく「自分のdfを問いに答えられる形に整える作業」として触る。
  - 例：`freq_drop = workout_freq_m1 - workout_freq_m3` を作る、層フラグを列にする。
- 当日の問い：**「あと一押し層」を、コードで再現可能な1つのbool列にできるか？**
- ※ Phase2では条件文（言葉）だった定義を、**実装された列**に昇格させる。

### スプリント1：現状を正しく描く（コマ15〜17）
- 目的：経営層に見せる「1枚」を**実データのグラフ**にする（Q02の結論文の根拠を可視化）。
- 学生が自分で決める：何の分布を見るか／どの軸で層別するか／どのグラフ種が伝わるか。
- コマ15＝数字で輪郭（記述統計・分位）、コマ16〜17＝**伝わる可視化を自分で設計**。
- 詰まったら見るヒント：「比較したいなら箱ひげ/バイオリン」「割合ならstacked bar」程度のカタログのみ。
- 当日の問い：**あと一押し層と継続層は、どの変数で一番くっきり分かれるか？**

### スプリント2：その差は本物か（コマ18〜21）
- ここで初めてテキスト（Ch8の検定の考え方）を**辞書として**差し込む。
- 前処理は「教科書的に網羅」ではなく「**自分のdfの汚れ**」に対して必要な分だけ。
  - 欠損・外れ値・スケーリングを「なぜ今これが要るか」を言わせる。
- 検定：スプリント1で見つけた差を、二群比率の検定 / t検定 / カイ二乗で確かめる。
- **選択バイアス**を正面から：`push_opt_in` は自己選択 → 「相関であって因果ではない」を必ず書く。
- 当日の問い：**スプリント1で見えた差は、偶然では説明できないか？因果と言えるか？**

### スプリント3：予測して、説明する（コマ22〜26）
- ここが経験者向けの本丸。ただし**狙いは予測精度ではなく「誰に効くか」の説明**。
- 分類（継続/離脱）または回帰（継続日数）を、**問いに合う方を学生が選ぶ**。
- 評価：accuracyではなく、**問いに合う指標**（離脱検知ならRecall/PR）を選ばせる。
- 特徴量重要度・SHAP：モデルを「**施策ターゲットの言語**」に翻訳する道具として使う。
  - 「SHAPでlogin_rate_m1の寄与が大きい → だから1ヶ月目に介入」まで降ろす。
- アンサンブル（LightGBM等）は「精度のため」ではなく「重要度を安定させるため」程度の位置づけ。
- 当日の問い：**モデルは『あと一押し層に何をすべきか』を新しく教えてくれたか？**

### スプリント4：計画を結論に変える（コマ27〜28）
- `analysis_plan.md`（計画）→ `analysis_report.md`（**実行済みレポート**）に昇格。
- Phase2で計画段階だったROIを、**実データで出た継続率改善見込み**で再計算。
- Q02の「結論文（仮）」を、分析結果で**確定文**に書き換える。
- コマ28は相互レビュー＋Phase4への橋渡し（教科書Ch6〜8での回収予告）。
- 当日の問い：**経営会議で『Go』と言わせられる1枚になったか？**

## 5. 学生主導にするための共通フォーマット（各コマ資料の中身）

完成コードを配らない代わりに、各コマはこの4ブロックだけにする：

```markdown
## 今週の問い
（analysis_plan.md の §X に対応する1問）

## 使えそうな手法カタログ（選ぶのは君だ）
- 手法A … こういう問いに向く / 向かない
- 手法B … 〃
（手順は書かない。選択基準だけ）

## 詰まったら見るヒント
- ◯◯で詰まったら → △△の関数 / テキストの□節
- エラーの定番 → …

## 提出と相互レビュー
- 提出物：…
- レビュー観点：「なぜその手法か」「So What? に耐えるか」
```

> ねらい：手順を消すことで「**何をすればいいか自分で決める**」状況を強制する。
> 経験者なので実装はできる。足りないのは「選ぶ・解釈する」経験。そこだけを鍛える。

## 6. Phase2 / Phase4 とのつながり（壊さない）

- **Phase2 → 3**：計画書がそのまま作業指示書になる。§2問い・§5手法選定が毎スプリントの起点。
- **3 → Phase4**：Phase4は教科書Ch6〜8での「やったことの理論回収」＋House Prices。
  - Phase3で「手を動かして詰まった経験」があるほど、Ch6〜8の回収が刺さる。
  - なので**Phase3ではあえて理論を深掘りしすぎない**（Phase4の取り分を残す）。
