# Phase3 レポート：自分でデータ分析の事例を立ち上げる

> Phase3では **FitLoop という"用意された題材"** を分析して投資委員会で予算を勝ち取った。
> このレポートでは立場を逆にする。**題材そのものを自分で発想する**。
> 「世の中（または身近）にある、データ分析で意思決定を変えられそうな事例」を1つ選び、
> **問い → 仮説 → 提案 → 必要なデータ** までを自分の頭で設計する。
> データを実際に集めて分析する必要はない。**「どう考えるか」の筋道を書ききる**のがゴール。
>
> 関連：[[phase3_summary]]（必要な考え方）／[[12_your_question_1]]（問いの設計）／[[07_analysis_story]]（So What?）

---

## このレポートのポイント

手を動かす量ではない。次の3つが見られている。

1. **問いが「決定に効く形」になっているか**（知っても何も変わらない問いはNG）
2. **観察→So What?→提案** がつながっているか（数字の先に行動があるか）
3. **正直さ**：自分の案の弱点（バイアス・データの欠け）を、聞かれる前に自分で書けるか

> Phase3のルーブリックと同じ思想。L1（問い1本＋根拠1つで語れる）→ L2（数字で
> Go/No-Go＋バイアス自覚）→ L3（弱点を踏まえて提案を組み替えられる）。

---

## 書き方の流れ（全10ステップ）

下の **1〜9 を埋め、最後に 10 で振り返る**。
各セクションは「**問い**（あなたが答えること）／**記入欄**／**ヒント**／**薄い記入例**」の順。
記入例は「学食の混雑」を題材にした薄いサンプル。**マネせず、自分の題材で書くこと**。

---

### 1. 自分で考えたテーマ

**問い：どんな場面の、誰の、どんな困りごとを扱うのか？**

```
（記入欄）
テーマ名：
場面・対象：
困っている人：
```

**ヒント**
- 身近でよい（バイト先・部活・学食・通学・SNS・ゲーム・地域の店…）。
- 「面白いか」より「**誰かが判断を間違えていそうか**」で選ぶと、後がラク。
- 大きすぎる社会問題（例：少子化）は避ける。**手触りのある単位**に落とす。

> 記入例：学食が12:15〜12:45に激混みし、学生は並ぶのを諦めて外に流出。
> 困っている人＝学食の運営者（売上を取りこぼしている）。

---

### 2. 現状あると思われるデータ（＝いま既にあるはず）

**問い：その現場に、今この瞬間どんなデータが転がっているか？**

```
（記入欄）
今ありそうなデータ：
- 
- 
- 
誰が持っている：
形（紙／レジ／アプリ／Excel／記録なし…）：
```

**ヒント**
- **「理想のデータ」ではなく「現実にありそうなデータ」**を書く。ここがズレると絵に描いた餅になる。
- 「記録されていなさそう」も立派な発見。後の 7 で効いてくる。

> 記入例：レジの会計時刻ログ（売上時刻はある）／券売機の販売数。
> 一方、**「並んで諦めて帰った人」は記録に残らない**（＝見えない需要）。

---

### 3. 何を目的とするか

**問い：この分析で、誰のどんな決定を変えたいのか？**

```
（記入欄）
目的（誰の・何の決定を・どう変えたい）：
これが分かると変わること：
```

**ヒント**
- 「〇〇を知りたい」で止めない。「**知った結果、何をするか**」まで書く。
- 目的＝**意思決定**。分析は決定の道具。決定が見えない目的はやり直し。

> 記入例：運営者が「**増員・メニュー絞り込み・時間分散**のどれに手を打つか」を
> 根拠を持って選べるようにする。

---

### 4. よい問いとは何か（＋自分の問い）

**問い：意思決定に効く問いを、一文で立てられるか？**

まず「よい問い」のチェックリストで自分を採点する：

- [ ] **具体的**か（対象・期間・指標が一文に入っている）
- [ ] **測れる**か（数字で答えが出る形か）
- [ ] **決定に効く**か（答えがYes/Noで打ち手が変わるか）
- [ ] **手が届く**か（現実にありそうなデータで答えられそうか）

```
（記入欄）
私の問い（一文）：
　例の型 →「〇〇（条件）の人は、△△（指標）が□□と関係しているか？」
この問いがよい理由（上のどれを満たすか）：
```

**ヒント**
- ダメな問い＝「**学食を良くするには？**」（漠然・打ち手が決まらない）。
- よい問い＝「**11:45より前に来る学生**は、**外部流出**と関係があるか？」（条件・指標・打ち手が見える）。
- 迷ったら「**この問いの答えがYesとNoで、運営者の行動は変わるか？**」を自問。変わらないなら問いが弱い。

> 記入例：**売上ピークの時間帯**と**1人あたり会計時間**には関係があり、
> "会計の遅さ"が流出を生んでいるか？

---

### 5. 仮説（答えの予想）

**問い：その問いに、今の時点でどう答えると予想するか？なぜ？**

```
（記入欄）
仮説（〜だろう）：
そう考える根拠（経験・観察・常識）：
外れていたら何が言える：
```

**ヒント**
- 仮説は**当てにいくものではない**。「**外れたら何が分かるか**」まで書けると強い。
- 「たぶん混んでるから」で終わらせず、**メカニズム**（なぜそうなるか）を一言添える。

> 記入例：ピークの流出は"席不足"より"**会計列の詰まり**"が主因だろう。
> 根拠＝外で待つ列は券売機前に集中している印象。外れれば席数が主因と分かる。

---

### 6. 提案（So What? の先：誰に・何を・いつ）

**問い：仮説が正しかったら、どんな打ち手を提案するか？**

```
（記入欄）
提案（誰が・何を・いつ）：
期待できる効果（できれば数字の見込み）：
この提案のコスト・制約：
```

**ヒント**
- Phase3と同じ型：**観察（事実）→ So What?（だから何）→ 行動（提案）** を一本につなぐ。
- 「分析しました」で終わるのが一番もったいない。**提案がレポートの主役**。
- コスト・制約を書くと、提案が一気に現実的になる（FitLoopの"予算40万"と同じ発想）。

> 記入例：11:45〜12:00に**事前決済（モバイルオーダー）**を導入し会計列を短縮。
> 流出の1〜2割を回収できれば導入コストを上回る見込み。制約＝券売機の改修費。

---

### 7. そのために必要なデータ（＝いま無いが、取りに行くべき）

**問い：問いに本当に答えるには、2 に足して何のデータが要るか？**

```
（記入欄）
追加で必要なデータ：
- （何を）　　　（どう取るか：計測／アンケート／観察／ログ追加…）
- 
取れない／取りにくいデータと、その代わりの工夫：
```

**ヒント**
- ここが今回の**山場**。2の「現状あるデータ」と、問いに答えるのに**本当に要るデータ**の差分を埋める。
- **見えない需要**（諦めた人・買わなかった人）をどう拾うかを必ず考える。
- 「全部は取れない」が普通。**取れない時の代理指標（プロキシ）**を1つ考えると評価が上がる。

> 記入例：諦めた人数は記録に無い → **昼の10分だけ入口で人数カウント**で代用。
> 会計時間は**ストップウォッチで20人サンプル計測**すれば概算できる。

---

### 8. 分析の段取り（どう確かめるか）

**問い：集めたデータで、問いをどう検証するつもりか？**

```
（記入欄）
見る指標／比べる軸：
使えそうな手法（1つでよい・選んだ理由つき）：
「この結果が出たらYes、出たらNo」の判定ライン：
```

**ヒント**
- 手法は**1つで十分**。「比較するだけ」「相関を見るだけ」でも、**選んだ理由**が言えれば合格。
- Phase3で触れた引き出し：群の比較（検定）／相関・回帰／予測モデル＋重要度。**問いに合うものだけ**。
- **判定ライン**を先に決めておくのがプロ。「〇〇分以上なら会計が主因と判断」のように。

> 記入例：ピーク帯と非ピーク帯で**1人あたり会計時間を比較**。
> 平均で30秒以上の差があれば"会計の詰まり"を主因とみなす（群の比較）。

---

### 9. 落とし穴（自分の案の弱点を、自分で撃つ）

**問い：この分析・提案を、賢い人が反論するとしたら何と言うか？**

```
（記入欄）
弱点①（例：選択バイアス）：
弱点②（例：サンプルが少ない／期間が偏る）：
弱点③（例：相関であって因果ではない）：
それでも価値がある理由：
```

**ヒント**
- ここが **L2→L3の分かれ目**。弱点を隠す人は撃墜され、**先に自分で言う人**が信頼される。
- 定番の3つ：**選択バイアス**（測れた人が偏ってる）／**因果と相関の混同**／**サンプルの小ささ・期間の偏り**。
- 「弱点がない」は書けない（必ずある）。**正直に1つ以上**挙げること。

> 記入例：入口カウントは"諦めた人"と"通りすがり"を区別できない（測定バイアス）。
> 1日だけの計測では曜日差を拾えない。→ 複数曜日×複数回で粗さを補う。

---

### 10. 振り返り（このレポートの本体）

**問い：自分で題材から考えてみて、何が難しく、どんな工夫が要ると感じたか？**

> ここが最重要。**正解を出すこと**ではなく、**考える過程で何にぶつかったか**を言語化する。
> 200〜400字程度で、下の観点から2つ以上に触れて書く。

- どのステップが一番難しかったか（テーマ選び？問いの一文化？データの差分？弱点出し？）
- なぜ難しかったか（言葉にしようとすると曖昧になる／データが無いと気づいた…）
- もし本当にやるなら、どんな工夫・段取りが要りそうか
- FitLoop（用意された題材）を分析した時と、**自分で立ち上げた今回**で、難しさはどう違ったか

## 提出前セルフチェック

- [ ] 1〜10 がすべて埋まっている
- [ ] 4 の「私の問い」が**一文**で、YesでもNoでも打ち手が変わる
- [ ] 6 の提案が「**誰が・何を・いつ**」の形になっている
- [ ] 7 で「現状データ」と「必要データ」の**差分**を埋めた
- [ ] 9 で自分の弱点を**1つ以上**正直に書いた
- [ ] 10 の振り返りが、**感想ではなく"どこが難しくどう工夫するか"**になっている

## 採点の目安（自己採点にも使える）

| 段 | 状態 | 目安 |
|---|---|---|
| **L1** | 問いが一文で立ち、根拠の見込みが1つ言える | 〜60% |
| **L2** | 観察→提案がつながり、必要データと弱点を自分で書けた | 〜85% |
| **L3** | 弱点を踏まえて提案や問いを"こう組み替える"まで書けた | 85%〜 |

> 分量で稼ぐレポートではない。**問いの鋭さ・提案までの一本道・弱点の正直さ**で決まる。
