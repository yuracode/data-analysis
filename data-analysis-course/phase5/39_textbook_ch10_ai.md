## 本編

### セクション1：AIと機械学習の現在地

```
AI
└── 機械学習（ML）
    ├── 教師あり学習（分類・回帰）← このコースで学んだ範囲
    ├── 教師なし学習（クラスタリング・次元削減）
    └── 強化学習（ゲーム・ロボット制御）
        └── 深層学習（DNN・CNN・Transformer）← TF経験あり
```

現在のMLが「できること」と「できないこと」：

| できること | できないこと |
|-----------|------------|
| パターン認識（画像・テキスト） | 因果推論（なぜかは説明できない） |
| 大量データからの予測 | 分布外のデータへの汎化 |
| 人間より速く・安定して処理 | 常識・文脈の理解 |
| 特徴量の重要度を算出 | 目的の設定（何を解くべきか） |

### セクション2：MLモデルの限界

**1. 分布シフト（Distribution Shift）**
```
学習データと運用データの分布が異なる場合、精度が劣化する

例：コロナ禍での消費行動の急変 →
2019年データで学習したモデルが2020年に機能しなくなった
```

**2. 公平性（Fairness）**
```
訓練データに含まれる偏り（バイアス）がモデルに引き継がれる

例：採用モデルが過去のデータから「男性が採用されやすい」を学習
→ 意図せず性別による差別をするモデルが生まれる
```

**3. ブラックボックス問題**
```
高精度なGBM・ニューラルネットは「なぜその予測か」が説明しにくい

対処：SHAP, LIME, PDP（Phase 4 で実施済み）
規制：EUのAI法では一部の高リスクAIに説明可能性を義務化
```

### セクション3：「AIが使える問い」の条件

```
1. 繰り返し発生する問い（1回しかない問いには使えない）
2. 大量のデータがある（少ないデータでは学習できない）
3. 問いが明確に定義できる（「なんとなく改善したい」は難しい）
4. 誤りのコストが許容範囲内（医療・法律の最終判断は慎重に）
```

## 今日のまとめ
- MLは強力だが「データに基づく相関モデル」であり万能ではない
- 分布シフト・公平性・説明可能性は実運用で必ず向き合う課題
- 「AIを使う」判断より「どのリスクを許容するか」の判断が重要
