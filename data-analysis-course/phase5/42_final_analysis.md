最終課題の分析は「Phase 1〜4の習得事項のアウトプット」として位置づける。

推奨ワークフロー：

```
1. データ読み込み・EDA（30分）
   └── 形・欠損・分布・相関を確認

2. 前処理・特徴量エンジニアリング（30分）
   └── ColumnTransformer + Pipeline を使う

3. ベースラインモデル（15分）
   └── まずシンプルなモデルでCVスコアを確認

4. モデル改善（15分）
   └── LightGBMまたはGBM + 特徴量追加

5. 解釈・考察（残り時間）
   └── SHAP または 統計検定で問いに答える
```

### セクション2：詰まったときのガイド

| 症状 | 対処 |
|------|------|
| 欠損が多すぎる | ドメイン知識で「意味のある補完」を考える |
| CVスコアが改善しない | 特徴量エンジニアリングを追加してみる |
| 目的変数が偏っている | `class_weight='balanced'` or オーバーサンプリング |
| 過学習している | 正則化・特徴量削減・早期停止 |
| SHAPが動かない | `shap.Explainer(model, X_train)` で試す |
