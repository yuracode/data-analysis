## 本編

### セクション1：レポートの目的と読者を定める

良いレポートは「誰が読むか」から逆算して設計する：

| 読者 | 関心事 | レポートの焦点 |
|------|--------|--------------|
| 経営者・意思決定者 | 何をすべきか | 結論・提言・ビジネスインパクト |
| 現場マネージャー | どう動くか | 具体的なアクションと優先順位 |
| データサイエンティスト | 手法が正しいか | 前処理・モデル・評価の詳細 |

> 同じ分析でも「誰向けか」によって書くべき内容が変わる

### セクション2：エグゼクティブサマリーの構造

```markdown
## エグゼクティブサマリー

### 背景
（分析の目的・なぜやったか：2文以内）

### 主な発見
- 発見1：（数値を含めて一文で）
- 発見2：
- 発見3：

### 提言
1. （優先度最高）〇〇を実施する
2. 〇〇を検討する

### 期待される効果
（KPIの数値で表現）

### 限界・注意点
- この分析では因果関係は示せていない
- サンプルサイズ：N件
```

### セクション3：数字を「意味」として語る

悪い例と良い例：

```
悪い例：「モデルの精度は83%でした」

良い例：「モデルの精度は83%です。これは100人の学生のうち17人について
誤った予測をすることを意味します。特に就職できない学生を「就職できる」と
誤予測する（偽陽性）ケースが○件あり、支援の機会損失が生じる可能性があります。」
```

ストーリーテリングの型：
```
1. 問い（What）：何を明らかにしたかったか
2. 発見（So What）：何がわかったか
3. 提言（Now What）：だから何をすべきか
4. 根拠（Why Trust）：なぜ信頼できるか（手法の妥当性・限界）
```
