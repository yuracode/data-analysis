### 評価ルーブリック（合計20点）

| 評価項目 | 5点 | 3点 | 1点 |
|---------|-----|-----|-----|
| **問いの質** | SMARTを満たし、ビジネス・社会的意義が明確 | 問いは具体的だが意義の説明が弱い | 問いが曖昧または分析対象として測定不能 |
| **分析の適切さ** | 問いに合う手法を選択し、前処理・モデルが妥当 | 手法は実装できているが選択理由が弱い | 問いと手法がズレている／前処理にリーク等の重大な問題 |
| **解釈・考察の深さ** | 数値をビジネス文脈に翻訳し、限界も明記 | 結果を説明できるが「だから何か」の踏み込みが浅い | 数値の羅列のみ／提言なし |
| **伝わるか** | Notebookとスライドだけで第三者が理解できる | 大筋は伝わるが補足説明が必要 | 説明不足／グラフのラベル欠落／論理が飛躍 |

## 本編

### セクション1：テーマ選定の基準

良いテーマの条件：

```
1. データが入手可能
   - Kaggle公開データセット
   - 政府統計（e-Stat等）
   - UCI ML Repository

2. 問いが明確
   - 「〇〇を予測したい」「〇〇の要因を特定したい」
   - SMARTな条件を満たす

3. ビジネス・社会的意義がある
   - 「なぜこれを知りたいか」を1文で言える

4. Phase 3 の手法で対応できる
   - 分類・回帰・クラスタリング等
```

### セクション2：テーマ候補の例

| テーマ | 問い | 手法 |
|--------|------|------|
| 心臓病リスク予測 | 医療データで発症リスクを予測できるか | 分類 |
| 中古車価格予測 | 車種・走行距離等から価格を予測できるか | 回帰 |
| 顧客離反予測 | 離反しそうな顧客を事前に特定できるか | 分類 |
| 都市の犯罪分析 | 時間帯・地域で犯罪発生のパターンはあるか | EDA・統計 |
| 農作物収穫量予測 | 気象データから収穫量を予測できるか | 回帰 |
