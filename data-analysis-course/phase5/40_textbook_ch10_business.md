## 前回の振り返り

## 本編

### セクション1：MLプロジェクトのライフサイクル

```
1. 問いの定義
   → KPI・評価指標・成功基準を決める

2. データ収集・前処理
   → 品質・量・代表性を確認する

3. モデリング（PoC）
   → 「技術的に実現可能か」を最短で検証する

4. 評価・意思決定
   → ビジネス価値（ROI）と技術的リスクのバランス

5. 本番化・モニタリング
   → 精度劣化・分布シフトを継続的に監視する

6. 改善サイクル
   → 新データで再学習・モデルの更新
```

### セクション2：PoC の設計

**PoC（Proof of Concept）**：「このアイデアは機能するか」を最小コストで検証する

```markdown
## PoC設計書テンプレート

### 検証したいこと
（例：過去データで就職可否を予測できるか）

### 成功基準
（例：CVスコア AUC > 0.75）

### 期間・コスト
（例：2週間・担当1名）

### 判断基準
- 成功 → 本番化の検討へ
- 失敗 → 問いを変えるか、データ追加の検討

### リスク
（例：データ不足・目的変数の定義が曖昧）
```

### セクション3：データ分析者の価値

データ分析者が生み出す価値のレイヤー：

```
Level 1（データの処理）：正確に集計・可視化できる
Level 2（仮説の検証）：問いを立てて統計・MLで検証できる
Level 3（意思決定の支援）：結果をビジネス提言に変換できる
Level 4（問いの設計）：組織が気づいていない問いを発見できる
```

このコースで目指しているのは **Level 3〜4**。

- Level 1・2は「ツールの習熟」で到達できる
- Level 3・4は「ビジネス理解」×「コミュニケーション」が必要
- Phase 2 で学んだ「問いの設計」こそが Level 4 の核心
