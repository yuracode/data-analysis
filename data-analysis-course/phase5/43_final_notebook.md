## 本編

### セクション1：Notebookの最終整備チェック

```
□ Restart & Run All で全セルが通る
□ 各セクションの前にMarkdownで「目的」「解釈」を書いた
□ すべてのグラフにタイトル・軸ラベルが付いている
□ エグゼクティブサマリー（冒頭セル）を書いた
□ 最終結論と提言を末尾のセルに書いた
□ コードの試行錯誤部分を削除した
□ 重要なパラメータはコメントで理由を書いた
```

### セクション2：発表スライドの構成

推奨スライド構成（発表時間：5〜10分）：

| スライド | 内容 | 枚数目安 |
|---------|------|---------|
| 1. タイトル | テーマ・名前 | 1 |
| 2. 問いと背景 | なぜこの問いか | 1 |
| 3. データ | データセットの説明 | 1 |
| 4. EDAの主な発見 | 可視化2〜3枚 | 2 |
| 5. モデルと評価 | 手法・CVスコア | 1 |
| 6. 主な発見・洞察 | 特徴量重要度・SHAP | 1〜2 |
| 7. 提言 | What should we do? | 1 |
| 8. 限界・今後の展望 | | 1 |

合計：9〜10枚

### セクション3：発表で意識すること

```
□ 最初に「何を明らかにしたかったか」を30秒以内に伝える
□ グラフは「1スライド1メッセージ」
□ 数字はそのまま出さず意味に翻訳する
□ 「自分の分析の限界」を自分で言える（信頼性が上がる）
□ 聴衆が「だから何？」と思わないように提言まで話す
```
