## シナリオ

あなたは **XX 専門学校 ICT 学科** の教務委員会の一員として、来年度（2026年度）の **学習・就職支援プログラム** を経営層に提案することになった。

経営層からの依頼は以下のとおり。

> 「今年度より**就職の質**を改善したい。
>  ただし、追加で投入できる予算と人員は限られている。
>  **どの学生層**に、**どんな支援**を、**どのタイミング**で行うべきかを、
>  過去3年分の学生データから根拠を示して提案してほしい。」

教務委員長からの追加コメント：

> 「『就職率を上げる』だけだと、ザルなKPIになる。
>  GPAが高い学生は放っておいても決まる。本当に効くのは『あと一押しで変わる層』への介入だ。
>  そこをデータで切り出してほしい。」

このシナリオに対し、Phase2 を通じて分析計画を組み立てていく。

## 共通データ仕様

各コマで同じ `df` を使い続ける。コマ8の冒頭で1度生成し、以降は再利用する。

```python
import pandas as pd
import numpy as np
import matplotlib
matplotlib.rcParams['font.family'] = 'IPAGothic'
import matplotlib.pyplot as plt

RANDOM_STATE = 42
rng = np.random.default_rng(RANDOM_STATE)

n = 600  # 過去3年 × 約200人/年
df = pd.DataFrame({
    'student_id':     np.arange(1, n + 1),
    'entry_year':     rng.choice([2022, 2023, 2024], n, p=[0.32, 0.34, 0.34]),
    'department':     rng.choice(['ICT', '情報処理', 'AI', 'セキュリティ'], n),
    'gpa_y1':         np.round(rng.normal(2.7, 0.6, n).clip(0, 4), 2),
    'gpa_y2':         np.round(rng.normal(2.9, 0.6, n).clip(0, 4), 2),
    'attendance_y1':  np.round(rng.beta(8, 2, n), 2),
    'attendance_y2':  np.round(rng.beta(8, 2, n), 2),
    'cert_count':     rng.poisson(1.5, n),
    'internship':     rng.binomial(1, 0.45, n),
    'extra_program':  rng.binomial(1, 0.35, n),   # 補講プログラム参加フラグ
    'got_job':        rng.binomial(1, 0.70, n),
})

# 「効くはずの変数」を弱く埋め込む（分析する価値が出るように）
boost = (
    0.12 * (df['attendance_y1'] > 0.85).astype(int)
    + 0.10 * (df['extra_program'] == 1).astype(int)
    + 0.08 * (df['cert_count'] >= 2).astype(int)
)
df['got_job'] = ((df['got_job'] + rng.binomial(1, boost.clip(0, 1))) >= 1).astype(int)
```

> **注意**：このデータは仮想生成。学生に「実データのつもりで分析するが、結論を実校に持ち込まない」ことを明示する。

## コマ6 担当パーツ：問いの初期設計

### お題

経営層・教務委員長のコメントを、**5個の小さな問い**に分解せよ。

### やること

1. 経営層のメッセージから「KPI候補」を**3つ以上**挙げる
   （就職率？／内定までの早さ？／早期離脱の防止？／質の高い就職先比率？など）
2. ロジックツリーで大きな問いを 5 個の小問いに分解する（Mermaid または テキスト可）
3. 各小問いに対して、以下を1行で書く：
   - 答えが出たら **何が変わるか**（So What?）
   - **答えやすさ**（◎ ／ ○ ／ △ ／ ×）
   - **意思決定への効きやすさ**（◎ ／ ○ ／ △ ／ ×）
4. 5個の中から、Phase3 以降で深掘りする **1個** を仮選定する

### 検討させる視点

- 「就職率を上げる」を**そのまま**KPIにすると何が困るか？
- 問いを **個人レベル / セグメントレベル / プログラム全体** のどこに置くかで、必要なデータと手法が変わる
- 教務委員長の「あと一押しで変わる層」は、どの問いに対応するか？

### 成果物パーツ

- `Q01_question_list.md`：問いリスト（5個）+ 仮選定 1個 + 選定理由（3行）

## コマ7 担当パーツ：分析ストーリーの設計

### お題

仮選定した問いに対する **「経営会議で見せる最終1スライド」** をテキストで設計せよ。

### やること

1. 結論文を1文で書く（仮でよい、分析後に書き換えてOK）
   - 例：「就職率向上には、1年次後期に出席率70%未満となった学生への補講介入が最も費用対効果が高い」
2. その結論を支える根拠を **3つ** 用意する（どんな数字・グラフが出れば結論が言えるか）
3. 想定される **反論を2つ** 挙げ、それぞれへの応答方針を書く
4. ピラミッド構造で全体を1枚にまとめる（テキスト or Mermaid）

### 検討させる視点

- 「数字 → 意味 → 施策」の3層が揃っているか？
- 結論が「だから何？（So What?）」に耐えるか？
- 経営層が **最初に知りたいのはどの順番か**（時系列 vs 結論ファースト）？

### 成果物パーツ

- `Q02_storyboard.md`：結論文 + 根拠3つ + 想定反論2つ + ピラミッド図

## コマ8〜9 担当パーツ：学生セグメンテーション

### お題

「あと一押しで変わる層」を**データで定義**せよ。

### やること

1. 以下の手法候補から **2つ以上** を実際に試し、結果を比較する

   | 手法候補 | このプロジェクトでの解釈 | 学べるコマ |
   |---------|----------------------|----------|
   | RFM 分析 | R=出席率, F=GPA, M=資格数 として3軸スコア化 | コマ8 |
   | コホート分析 | 入学年度 × 学年 で就職率の経時変化 | コマ8 |
   | ピボット集計 | 学科 × 出席率帯 × 就職率 など多変量集計 | コマ9 |
   | （発展）k-means | 多変量グルーピング（Phase3 で再検討可） | Phase3 |

2. 「あと一押し層」の **定義文** を1〜2行で書く
   （例：`attendance_y1 ∈ [0.6, 0.8]` かつ `cert_count == 1` かつ `extra_program == 0` の学生）
3. その層が全体の **何%** いるか、**就職率は全体平均と比べてどうか** を数値で示す
4. このセグメントへの **初期施策案** を1つ書く

### 検討させる視点（手法選定の根拠）

| 検討ポイント | 学生が答えるべきこと |
|------------|-------------------|
| なぜその手法を選んだか | 問い（=あと一押し層の特定）に対する適合性 |
| 他の手法を選ばなかった理由 | データ・粒度・解釈しやすさのどれが効いたか |
| セグメント数の決め方 | 経営層に説明できる粒度か？（多すぎ／少なすぎ） |
| RFM のスコア区切り | 分位数 vs 固定閾値、どちらが今回の問いに合うか |

### 成果物パーツ

- `Q03_segmentation.ipynb`：手法2つの実装 + セグメント定義 + 数値根拠 + 初期施策案

## コマ10〜11 担当パーツ：時系列傾向と施策の効果検証

### お題

セグメンテーションで定義した「あと一押し層」に対する施策の **効果検証計画** を設計せよ。

### やること

1. **時系列の観点**：以下から1つ以上を実装する
   - 入学年度 × 就職率の推移（年度効果はあるか）
   - 1年次出席率と2年次GPA の関係（早期シグナルとして使えるか）
   - 補講プログラム参加 × 就職率 の年度別比較

2. **施策の効果検証**：A/B テスト設計書を書く
   - A群・B群の定義（誰を、どう割り当てるか）
   - 評価指標と判定基準（何%差で「効果あり」とするか）
   - 必要サンプルサイズの目安
   - **選択バイアス**への対処方針（既存の `extra_program` フラグはランダム割り当てではない点に注意）

3. 既存データで二群比率の検定を実施し、観察された差と p 値を報告する
   （観察データなので「相関であって因果ではない」点を必ず明記）

### 検討させる視点

- **時系列**で見るべき粒度は？（日次・月次・学期・年度）データ点数とのバランス
- **A/B テスト**で「やってはいけない設計」は何か？（事後割り当て・自己選抜・対照群なし）
- 「既に補講に参加した人」の効果と、「これから参加させる人」の予測効果は同じか？

### 成果物パーツ

- `Q04_timeseries_and_ab.ipynb`：時系列分析の結果 + A/B 設計書 + 既存データでの検定結果
