# [コマ12] 自分の問いを設計する①

## 本日の目標
- Phase 2で学んだ知識を総動員して「自分の問い」を設計できる
- ロジックツリーで問いを分解し、必要なデータを特定できる
- Phase 3以降の分析テーマの候補を1〜2個確定できる

## 前回の振り返り
- A/Bテストのシミュレーションを実装し、結果の解釈と提言の書き方を練習した

## 本編

### セクション1：「自分の問い」の条件確認

良い問いのチェックリスト：

- [ ] SMARTな条件を満たしているか（Specific・Measurable・Achievable・Relevant・Time-bound）
- [ ] 意思決定に役立つか（「知っていたら何かが変わるか？」）
- [ ] Phase 3で学ぶ手法で答えられそうか
- [ ] 使えるデータ（Kaggle・公開データ等）が存在するか

### セクション2：学生データを使った問い設計の例

**例1：就職支援の優先順位付け**
```
問い：GPA・出席率・資格数のどの組み合わせが最も就職率と関連しているか？
KPI：就職率（%）
手法候補：ロジスティック回帰・決定木
データ：学生の成績・出席・就職結果データ
```

**例2：成績低下の早期警告**
```
問い：1年次前期の出席率でそのまま卒業できるかを予測できるか？
KPI：卒業率・退学率
手法候補：ロジスティック回帰・Random Forest
データ：入学〜卒業の縦断データ
```

### セクション3：問い設計ワークショップ

以下のテンプレートで「自分の問い」を言語化する：

```markdown
## 私の問い

### 問い（一文で）
（例：〇〇のデータを使って△△がXXに与える影響を明らかにしたい）

### なぜこの問いか
（モチベーション・ビジネス・社会的意義）

### KPI
（何を測るか・単位）

### 仮説
（答えはこうなると予想する）

### 使うデータ
（Kaggle Dataset名 or 公開データのURL）

### 手法候補
（Phase 3で学ぶ予定の手法から選ぶ）
```

## 演習

### お題
上記テンプレートを使い、自分の問いを1つ設計する。

### やること
1. 問いのテンプレートを埋める
2. ロジックツリーを作成する（テキストで可）
3. ペアで発表し、お互いの問いに「So What?」を問いかけ合う

### 提出物
- 「自分の問い」テンプレート（記入済み）
- ロジックツリー

## ？リスト（疑問を記録しよう）
- 問いが複数浮かんだときの優先順位の付け方は？
- Kaggleにちょうどいいデータがない場合は？

## 今日のまとめ
- 「良い問い」はSMART条件を満たし、意思決定に直結する
- ロジックツリーで問いを分解すると「どのデータが必要か」が見えてくる
- 他者の問いに「So What?」を問いかけることで質が上がる
