## 授業全体を通じた注意点

- **個別コマの演習と通し課題は別物**  
  コマ6〜11 の個別演習ファイルは「概念を教えるための練習題材（学生データ等）」。  
  通し課題（FitLoop）は並行して積み上げる別のプロジェクト。  
  「今やっているのは概念の練習か、通し課題か」を毎コマ口頭で明示すること。

- **dfの生成はコマ8冒頭の1回のみ**  
  コマ9以降は同じ `df` を再利用。コマ8でNotebookを保存させ、以降は冒頭でloadする習慣をつけさせる。

- **財務感覚を常に意識させる**  
  「月次予算40万円・プラン別月額料金」という制約を毎回思い出させる。  
  セグメントや施策を決めるたびに「それはいくらかかるか」を問いかける。

### 時間配分（90分）

|------|------|

| 55〜85 | 通し課題着手（FitLoop Q01：KPI候補3つ＋ロジックツリー） |
| 85〜90 | まとめ・次回予告 |

### トークスクリプト（箇条書き）

**導入**
- 「データ分析で一番大事な仕事は何だと思いますか？コードを書くこと……ではなくて、"良い問いを立てること"です」
- 「問いが曖昧なまま分析を始めると、最後に『で、これって何に使えるの？』になります。今日はそれを防ぐ技術を学びます」

**SMART条件の説明時**
- 「SMARTの中で一番見落とされるのが M（Measurable）です。『顧客満足度を上げたい』はデータで測れますか？何の数字を見ますか？」
- 「T（Time-bound）がないと、いつまでに答えを出すのかが決まりません。分析の締め切りはビジネスの意思決定タイミングで決まります」

**KPI説明時**
- 「KGI（最終目標）とKPI（中間指標）の違いを覚えておいてください。KGIは『売上1億円』、KPIは『月間新規顧客数100件』。KPIが決まると分析のスコープが自動的に決まります」

**通し課題の導入**
- 「ここからFitLoopのシナリオに入ります。みなさんはFitLoopのプロダクト改善チームのメンバーです。月次予算40万円でユーザーを継続させる施策を考えてほしいという依頼が来ています」
- 「プレミアムプランのユーザーは月2,980円払っています。このユーザーが離脱すると、会社は何を失いますか？それを数字で意識しながら問いを立ててください」
- 「ヒント集の『タスク1』を見ながら、KPI候補を3種類の切り口で出してみてください。15分で3つ以上」

### 時間配分（90分）

| 85〜90 | まとめ・次回予告 |

### トークスクリプト（箇条書き）

**ロジックツリー説明時**
- 「ロジックツリーを作る目的は『分析のスコープを絞ること』です。木の全部を分析しようとすると死にます。最終的に1〜2本の枝に集中する」
- 「葉ノードに辿り着いたとき、そこに使えるデータがあるかを確認してください。データがない枝はいったん切る」

**So What? の説明時**
- 「分析結果が出たら必ず自分に聞いてください：『So What?』。それで会議室の何かが変わりますか？変わらないなら、問いを再設計です」
- 「『出席率と継続率に相関がある → So What? → だから1ヶ月目に介入すべき → So What? → だから予算をここに投下する』という3層まで降りて初めて分析に価値が出ます」

**通し課題の導入**
- 「Q02ではまず結論文を1文書いてもらいます。仮でいいです。分析が終わったら書き換えます。大事なのは『今時点でどこに賭けているか』を言語化すること」
- 「根拠を3つ用意するとき、現状確認・介入有効性・タイミングの3種類に分けて考えてください。ヒント集の『タスク2の表』を参考に」
- 「反論を2つ考えるとき、『方法論への疑問』と『解釈への疑問』を1つずつ用意すると強いストーリーになります」

## 補足：個別演習ファイルと通し課題の使い分け

各コマの個別授業ファイル（06〜13の演習）には学生データ等を用いた**概念練習用の題材**が含まれています。  
通し課題（FitLoop）とは別物です。授業の流れとしては：

1. 個別ファイルで概念を理解する（練習題材で手を動かす）
2. 同じコマ内で通し課題（FitLoop）に応用する

という2段構えになっています。  
「今は概念練習か、通し課題か」をその都度口頭で明示することで学生の混乱を防いでください。
